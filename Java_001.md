JAVA의 특징
1. 객체지향 (OOP)
    * 모든 것을 객체 기반으로 프로그램을 작성
    * 상속, 캡슐화, 다형성, 추상화 등의 개념을 지원

2. 독립성
    * WORA(Write Once, Run Anywhere) : 한번만 작성하면 어디서든 실행 가능이라는 특징이 있음
    * JVM(Java Virtual Machine) 운영체제에서 실행됨

4. 안정성
    * 가비지 컬렉션(Garbage Collection)을 통해 안정적인 메모리 관리 가능.
    * 예외 처리(Exception Handling)를 통해 프로그램의 안정성을 높임.

7. 방대한 라이브러리와 활발한 커뮤니티
    * 이미 만들어진 수많은 기능으로 편리한 개발과 활발한 자료 공유가 가능.

JAVA의 장점

1. 플랫폼 독립성
    * JVM(Java Virtual Machine) 덕분에 운영체제에 상관없이 실행 가능.
    * 한 번 작성한 프로그램을 여러 환경에서 사용가능.

2. 객체지향 프로그래밍 지원
    * 코드의 재사용성과 유지보수성이 높음.
    * 대규모 프로젝트 개발에 적합.

3. 안정성
    * 자동 메모리 관리(Garbage Collection)를 제공하여 메모리 누수를 줄여줍니다.
 
4. 풍부한 라이브러리
    * 이미 만들어진 수많은 기능으로 편리한 개발과 활발한 자료 공유가 가능.

6. 높은 시장 점유율
    * 기업용 시스템, 금융권, 공공기관 등에서 많이 사용되어 취업 기회가 많음


JAVA의 단점

1. 실행 속도가 상대적으로 느림
    * JVM을 거쳐 실행되므로 C, C++ 같은 컴파일 언어보다 느릴 수 있습니다.

2. 메모리 사용량이 큼
    * JVM과 가비지 컬렉션으로 인해 메모리 소모가 비교적 많습니다.

3. 코드가 길어질 수 있음
    * 같은 기능을 구현해도 Python 같은 언어보다 작성해야 하는 코드가 많음.
