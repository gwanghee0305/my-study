1\. DDL (Data Definition Language, 데이터 정의어)

\-> "구조(틀)를 정의하는 명령어"

\-> 데이터베이스의 전체적인 뼈대(테이블, 스키마 등)를 생성, 변경, 삭제할 때 사용



&#x20;\* 대표 명령어:

&#x20;  \* CREATE (데이터베이스나 테이블 생성)

&#x20;    

&#x09;   CREATE 테이블 명 (

&#x09;   컬럼명 데이터타입 \[DEFAULT 기본값] \[제약조건],

&#x09;   컬럼명 데이터타입 \[제약조건],

&#x09;   . . .

&#x09;   \[테이블 레벨 제약 조건]

&#x09;    );



&#x20;  \* ALTER (기존 테이블 구조 변경 - 컬럼 추가, 데이터 타입 변경 등)



&#x09;ALTER TABLE 테이블명 ADD (컬럼명 데이터타입 \[DEFAULT 기본값]);



&#x20;  \* DROP (테이블이나 데이터베이스를 완전히 삭제) 



&#x09;DROP TABLE 테이블명;



&#x20;  \* TRUNCATE (테이블 구조는 남겨두고 데이터만 빠르게 전체 삭제)



&#x09;TRUNCATE TABLE 테이블명;



&#x20;\* 실제 활용 상황 예시



&#x20;  \* 새로운 회원 정보를 저장하기 위해 새로운 회원 테이블을 처음 만들 때 사용

&#x09;	CREATE TABLE member; (

&#x09;		 컬럼명 데이터타입 \[DEFAULT 기본값] \[제약조건],

&#x09;   		 컬럼명 데이터타입 \[제약조건],

&#x09; 		  . . .

&#x09;		 \[테이블 레벨 제약 조건]

&#x09;   		 );

&#x20;  \* 서비스가 성장해서 "회원 가입일" **정보를 추가로 저장해야 할 때** 실행하여 기존 테이블 구조를 수정



&#x09;	ALTER TABLE member ADD signup\_date DATE;

2\. DML (Data Manipulation Language, 데이터 조작어)

\-> "테이블의 데이터를 삽입, 수정, 삭제 명령어"

\-> 테이블은 그대로 두고, 그 안에 담기는 실제 값들을 삽입, 수정, 삭제할 때 사용



&#x20;\* 대표 명령어:

&#x20;  \* INSERT (새로운 데이터 삽입)



&#x09;INSERT INTO 테이블명 VALUES (값1, 값2, 값3, ...);		--  ※ 테이블 정의 순서대로 모든 값을 빠짐없이 넣어야 함



&#x20;  \* UPDATE (기존 데이터 수정)



&#x09;UPDATE 테이블명

&#x09;SET    컬럼명1 = 변경할값1,

&#x20;      		 컬럼명2 = 변경할값2,

&#x20;    		 ...

&#x09;\[WHERE  조건식];			 --  ※ 조건절 생략 시 테이블 전체 행이 수정 됨



&#x20;  \* DELETE (특정 조건의 데이터 삭제)



&#x09;DELETE FROM 테이블명

&#x09;\[WHERE 조건식];			--  ※ 조건절 생략 시 테이블 내의 모든 데이터가 삭제 됨



&#x20;\* 실제 활용 상황 예시

&#x20;  \* 사용자가 회원가입 시 웹사이트에서 가입 버튼을 누르면, 새 회원 정보를 DB에 저장 함



&#x09;INSERT INTO member (name, email) VALUES ('김철수', 'chul@email.com');   



&#x20;  \* 사용자가 프로필 사진이나 비밀번호를 바꿀 때 수정 함



&#x09;UPDATE member SET password = 'new\_password' WHERE id = 1;



&#x20;  \* 회원이 탈퇴를 요청했을 때 해당 행(Row)을 삭제 함



&#x09;DELETE FROM member WHERE id = 1;

3\. DQL (Data Query Language)

\-> "데이터를 조회하는 명령어"

\-> 데이터베이스에 저장된 데이터를 사용자가 원하는 조건에 맞춰 필터링하고 가져오는 언어



&#x20;\* 대표 명령어:  

&#x20; \* SELECT (데이터 검색/조회)



&#x20; - 기본 작성법

&#x09;SELECT 컬럼명

&#x09;FROM 테이블명

&#x09;WHERE 조건

&#x09;GROUP BY 그룹화 기준

&#x09;HAVING 그룹 조건

&#x09;ORDER BY 정렬기준;



&#x20; - 실행 순서 : FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY

&#x20;

&#x20;\* 실제 활용 상황:



&#x20;-- 전체  직원 중 연봉이 50,000,000원 이상 받는 직원의 직원명, 급여, 연봉, 부서코드를 조회



SELECT EMP\_NAME 직원명, SALARY 급여, SALARY\*12 연봉, DEPT\_CODE 부서코드

FROM EMPLOYEE

WHERE SALARY\*12 >= 50000000;
